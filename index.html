Step 1
✓ Set up your ECS cluster

Before you can run containers, you need a cluster. An ECS cluster is a logical grouping of tasks and services.

What you'll need: An AWS account with free tier access. Navigate to the ECS console and click "Create Cluster."
Step 2
✓ Create a task definition

A task definition is like a blueprint for your container. It tells ECS which Docker image to use, how much CPU and memory to allocate.

{ "family": "nextwork-task", "containerDefinitions": [{ "name": "app", "image": "nginx:latest", "portMappings": [{ "containerPort": 80 }], "memory": 512, "cpu": 256 }] }
Step 3
✓ Deploy your service

Create a service from your task definition. Set desired task count to 1 and use the "Rolling update" deployment type.

Step 4
✓ Configure load balancing

Attach an Application Load Balancer to distribute traffic and enable health checks.

Step 5
✓ Clean up resources

Delete your service, cluster, load balancer, and target group. Check for lingering ENIs or security groups.
